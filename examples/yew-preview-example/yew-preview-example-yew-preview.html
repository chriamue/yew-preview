Getting Started

Four steps to add interactive component previews to your Yew application.

1. Add Dependency

Add yew-preview to your Cargo.toml and declare the feature flag:

Cargo.toml
[dependencies]
yew = { version = "0.23", features = ["csr"] }
yew-preview = { git = "https://github.com/chriamue/yew-preview" }

[features]
default = []
yew-preview = []

2. Write Your Component

Define a standard Yew function component as usual:

src/components/button.rs
use yew::prelude::*;

#[derive(Properties, PartialEq)]
pub struct ButtonProps {
    pub label: AttrValue,
    pub disabled: bool,
}

#[function_component(Button)]
pub fn button(props: &ButtonProps) -> Html {
    html! {
        <button disabled={props.disabled}>{ &props.label }</button>
    }
}

3. Add Preview Variants

Gate a preview module with the feature flag and register variants with create_preview!:

src/components/button.rs (continued)
#[cfg(feature = "yew-preview")]
mod preview {
    use super::*;
    use yew_preview::prelude::*;

    yew_preview::create_preview!(
        Button,
        ButtonProps { label: AttrValue::from("Click me"), disabled: false },
        (
            "Disabled",
            ButtonProps { label: AttrValue::from("Can't click"), disabled: true }
        ),
    );
}

4. Register and Run

Wire components into the preview app and run with Trunk:

src/main.rs
use yew_preview::{create_component_group, prelude::*};

let groups: ComponentList = vec![
    create_component_group!("Buttons", Button),
];

html! { <PreviewPage groups={groups} /> }

// then in the terminal:
// trunk serve

Interactive Previews

Interactive previews let you edit component props live in the browser — no recompile needed. Select the Interactive tab in the config panel to see controls for each declared arg.

Arg Types

VariantControlDescription
ArgValue::Text(String)text inputA free-form text input. Use for labels, URLs, or any string prop.
ArgValue::Bool(bool)checkboxA checkbox. Use for flags, toggles, and boolean props.
ArgValue::Int(i64)number inputA number input for an unconstrained integer.
ArgValue::IntRange(value, min, max)sliderA slider for a bounded integer. Ideal for sizes, counts with known limits.
ArgValue::Float(f64)number inputA decimal number input. Use for opacity, ratios, or fractional values.

Per-type Usage

ArgValue::TextA free-form text input. Use for labels, URLs, or any string prop.
// declare
("label", ArgValue::Text("Hello".to_string()))
// read in render closure
let label = get_text(args, "label");
ArgValue::BoolA checkbox. Use for flags, toggles, and boolean props.
// declare
("enabled", ArgValue::Bool(true))
// read in render closure
let enabled = get_bool(args, "enabled");
ArgValue::IntA number input for an unconstrained integer.
// declare
("count", ArgValue::Int(42))
// read in render closure
let count = get_int(args, "count");
ArgValue::IntRangeA slider for a bounded integer. Ideal for sizes, counts with known limits.
// declare
("size", ArgValue::IntRange(80, 1, 200))
// read in render closure
let size = get_int(args, "size") as u32;
ArgValue::FloatA decimal number input. Use for opacity, ratios, or fractional values.
// declare
("ratio", ArgValue::Float(0.8))
// read in render closure
let ratio = get_float(args, "ratio");

create_interactive_preview! (all types)

Use create_interactive_preview! when a component has only interactive variants — no static snapshots.

interactive only
use yew_preview::prelude::*;

create_interactive_preview!(
    MyComponent,
    args: [
        ("label",   ArgValue::Text("Hello".to_string())),
        ("enabled", ArgValue::Bool(true)),
        ("count",   ArgValue::Int(42)),
        ("size",    ArgValue::IntRange(80, 1, 200)),
        ("ratio",   ArgValue::Float(0.8)),
    ],
    |args| {
        let label   = get_text(args, "label");
        let enabled = get_bool(args, "enabled");
        let count   = get_int(args, "count");
        let size    = get_int(args, "size") as u32;
        let ratio   = get_float(args, "ratio");
        html! {
            <MyComponent
                label={label}
                enabled={enabled}
                count={count}
                size={size}
                ratio={ratio}
            />
        }
    }
);

Static Variants + Interactive

Populate both render and args in ComponentItem to get fixed snapshot tabs alongside the live Interactive tab.

manual Preview impl
// Static variants + interactive mode in one Preview impl
impl Preview for ImageComp {
    fn preview() -> ComponentItem {
        use std::rc::Rc;
        ComponentItem {
            name: "ImageComp".to_string(),
            render: vec![
                ("256".to_string(), html! { <ImageComp src={SRC} size={256u32} /> }),
                ("512".to_string(), html! { <ImageComp src={SRC} size={512u32} /> }),
            ],
            args: Some(InteractiveArgs {
                values: vec![
                    ("src".to_string(),  ArgValue::Text(SRC.to_string())),
                    ("size".to_string(), ArgValue::IntRange(256, 24, 1024)),
                ],
                render_fn: Rc::new(|args| {
                    let src  = get_text(args, "src");
                    let size = get_int(args, "size") as u32;
                    html! { <ImageComp src={src} size={size} /> }
                }),
            }),
            test_cases: vec![],
        }
    }
}

Native Server

ServePage
Default

Native Preview Server

Serve a component preview without trunk or WASM. The serve feature pre-renders every component variant via Yew SSR and serves the result with axum. No files on disk — all HTML is generated at startup.

Because it lives in dev-dependencies, axum and tokio are never included in your WASM production build.

1. Add the serve feature to dev-dependencies

The serve feature pulls in axum and tokio — only for native builds, never for the WASM production build.

Cargo.toml
[lib]
name = "my_components"
path = "src/lib.rs"

[dev-dependencies]
yew-preview = { ..., features = ["testing", "serve"] }

2. Export your component groups from a library target

The example file needs to import your groups, so expose them from lib.rs:

src/lib.rs
use yew_preview::prelude::*;
use yew_preview::create_component_group;

pub fn preview_groups() -> ComponentList {
    vec![
        create_component_group!("Buttons", Button::preview()),
        create_component_group!("Forms",   Input::preview()),
    ]
}

3. Add examples/serve.rs

Three lines — call serve_blocking with your groups and a port:

examples/serve.rs
fn main() {
    let port = std::env::var("PORT")
        .ok().and_then(|p| p.parse().ok()).unwrap_or(8080);
    yew_preview::serve_blocking(my_components::preview_groups(), port);
}

4. Run

No trunk, no wasm-pack. A native binary starts immediately:

terminal
cargo run --example serve
# yew-preview: http://localhost:8080

# different port:
PORT=9000 cargo run --example serve

Static Catalog

CatalogPage
Default

Static HTML Catalog

The catalog feature renders every component variant via Yew SSR and writes a single self-contained HTML file — a long-scroll catalog of all components, like a design-system reference sheet.

Because it lives in dev-dependencies, tokio and Yew SSR are never included in your WASM production build. The serve feature already implies catalog, so no extra setup is needed if you use both.

1. Add the catalog feature to dev-dependencies

The catalog feature pulls in tokio and Yew SSR — only for native builds, never for the WASM production build. If you already enable the serve feature, catalog is included automatically.

Cargo.toml
[lib]
name = "my_components"
path = "src/lib.rs"

[dev-dependencies]
yew-preview = { ..., features = ["testing", "catalog"] }

[[example]]
name = "catalog"
required-features = ["yew-preview/catalog"]

2. Export your component groups from a library target

The example binary imports your groups from the library target, so expose them from lib.rs:

src/lib.rs
use yew_preview::prelude::*;
use yew_preview::create_component_group;

pub fn preview_groups() -> ComponentList {
    vec![
        create_component_group!("Buttons", Button::preview()),
        create_component_group!("Forms",   Input::preview()),
    ]
}

3. Add examples/catalog.rs

Pass CatalogOptions to configure the output filename and inject your component CSS:

examples/catalog.rs
fn main() {
    let css = std::path::PathBuf::from(env!("CARGO_MANIFEST_DIR"))
        .join("my-styles.css");

    let output = std::env::var("OUTPUT")
        .ok()
        .map(std::path::PathBuf::from);

    let mut options = yew_preview::CatalogOptions::new("my-components")
        .css_file(css);

    if let Some(out) = output {
        options = options.output(out);
    }

    yew_preview::generate_catalog_blocking(
        my_components::preview_groups(),
        options,
    );
}

4. Run

No trunk, no wasm-pack. A native binary renders every component via Yew SSR and writes a single HTML file:

terminal
# Output: my-components-yew-preview.html (current directory)
cargo run --example catalog

# Custom output path
OUTPUT=dist/catalog.html cargo run --example catalog
